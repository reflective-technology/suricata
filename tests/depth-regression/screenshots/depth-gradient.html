結論：depth 是一個 threshold，不是修正。 這次調整把 1mb → 10mb 解掉了 2MB POST 的場景（90% → 0%）， 但同一條 keepalive 連線只要累積 toserver bytes 超過 10mb（例如批量上傳、長連線串小 POST）， bug 還是會回來，只是受影響比例變小。

根本解法還是回到 lua shape validation（攔截下游錯誤）或 fork Suricata 補 HttpGetRequestMethod wrapper（直接用 parser-validated method）。
